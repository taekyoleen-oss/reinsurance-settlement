# 재보험 정산 앱 설계서

> **목적**: 재보험 계약의 입력부터 정산, 미수/미지급금 관리까지 전 프로세스를 일관되게 처리할 수 있는 앱 화면 구성과 단계별 처리 절차를 정의합니다.
> **범위**: 비례재보험(Proportional Reinsurance)을 중심으로 하되, 비비례재보험(Non-Proportional)도 확장 가능한 구조로 설계합니다.

---

## 1. 재보험 정산 전체 프로세스 개요

재보험 정산은 출재사(Cedent)와 수재사(Reinsurer) 간 일정 주기(통상 분기 또는 반기)로 발생한 보험료·보험금·수수료를 상호 청산하고, 채권·채무를 정산하는 전 과정을 의미합니다 ([Munich Re](https://www.munichre.com/content/dam/munichre/contentlounge/website-pieces/documents/Proportional-Treaties-29-03-2023.pdf)).

### 1.1 대분류 프로세스 (End-to-End Flow)

```
[1단계] 계약 등록 (Treaty Setup)
   ↓
[2단계] 출재 명세 입력 (Cession Bordereau Input)
   ├─ 보험료 명세 (Premium Bordereau)
   └─ 손해 명세 (Loss Bordereau)
   ↓
[3단계] 기술계정 산출 (Technical Account)
   ├─ 재보험료 / 재보험금
   ├─ 출재수수료(Ceding Commission)
   ├─ 이익수수료(Profit Commission)
   └─ 적립금(Reserve Deposit) 및 이자
   ↓
[4단계] 계정명세서 작성 (Statement of Account, SOA)
   ↓
[5단계] 상호 확인 및 승인 (Confirmation)
   ↓
[6단계] 정산 및 송수신 (Settlement)
   ├─ 잔액 확정 (Balance Due)
   └─ 자금 이체 (Cash Movement)
   ↓
[7단계] 미수/미지급금 관리 (AR/AP Management)
   ↓
[8단계] 회계 마감 및 보고 (Closing & Reporting)
```

### 1.2 비례재보험 vs 비비례재보험 핵심 차이

| 구분 | 비례재보험 (Proportional) | 비비례재보험 (Non-Proportional) |
|---|---|---|
| 출재 기준 | 보험가입금액의 일정 비율 | 손해액 기준 (Excess Point 초과분) |
| 보험료 배분 | 원보험료 × 출재비율 | 재보험료율 × 대상 보험료 (별도 산정) |
| 보험금 배분 | 원보험금 × 출재비율 | 자기부담금(Excess) 초과분 |
| 수수료 | 출재수수료(Ceding Comm.) 존재 | 일반적으로 없음 |
| 정산 방식 | 정기 SOA 기반 | 손해 발생 시 청구(Cash Call) 중심 |
| 대표 형태 | Quota Share, Surplus | XOL, Stop Loss |

출처: [코리안리 재보험이해](https://koreanre.co.kr/publication/%EC%BD%94%EB%A6%AC%EC%95%88%EB%A6%AC%EC%99%80_%ED%95%A8%EA%BB%98%ED%95%98%EB%8A%94_%EC%9E%AC%EB%B3%B4%ED%97%98%EC%9D%B4%ED%95%B4.pdf), [Swiss Re 자료](http://newdoc.nccu.edu.tw/teasyllabus/308002308018/Swiss%20Re%20Prop.&Non-Prop..pdf)

### 2.2 [1단계] 계약(특약) 등록 화면

#### 2.2.1 화면 목적
재보험 거래의 기준이 되는 **특약(Treaty)**의 조건을 등록합니다. 이 정보는 이후 모든 정산 산출의 기초가 됩니다.

#### 2.2.2 입력 항목

**(1) 기본 정보**
| 필드명 | 형식 | 필수 | 설명 |
|---|---|:---:|---|
| 특약 코드 | Text(20) | ● | 내부 관리용 고유 식별자 (예: QS-2026-001) |
| 특약 명칭 | Text(100) | ● | 예: "2026년 화재보험 Quota Share Treaty" |
| 거래 형태 | Select | ● | 출재(Outward) / 수재(Inward) |
| 재보험 종류 | Select | ● | 비례 / 비비례 |
| 비례 세부 유형 | Select | ● | Quota Share / Surplus / QS+Surplus Combined |
| 보험종목(Class) | Select | ● | 화재 / 해상 / 자동차 / 배상책임 등 |
| 인수 기준 | Select | ● | 인수년도(UY) / 회계년도(Clean-Cut) |

**(2) 거래상대방 및 중개**
| 필드명 | 설명 |
|---|---|
| 거래상대방(Cedent/Reinsurer) | 마스터에서 선택 |
| 중개사(Broker) | 직접거래 시 공란 |
| 중개수수료율(Brokerage %) | 중개사 개입 시 |

**(3) 계약 기간 및 통화**
| 필드명 | 설명 |
|---|---|
| 계약 시작일 / 종료일 | 통상 1년 |
| 통화(Currency) | KRW / USD / EUR 등, 다중통화 가능 |
| 회계 주기 | 분기(Q) / 반기(SA) / 연간(A) |
| 정산 마감일(Due Date) | 회계기간 종료 후 N일 |

**(4) 비례재보험 핵심 조건**
| 필드명 | Quota Share | Surplus |
|---|---|---|
| 출재비율(Cession %) | 고정 (예: 30%) | 위험별 변동 |
| 보유액(Retention/Line) | - | 1 Line 금액 (예: 10억) |
| 한도(Limit/Lines) | - | N Lines (예: 9 Lines = 90억) |
| Treaty Capacity | 통상 없음 | 보유액 × Lines |

**(5) 수수료 체계 (Commission)**
| 필드명 | 설명 |
|---|---|
| 출재수수료율(Ceding Commission %) | 고정율 또는 Sliding Scale |
| Sliding Scale 구간 | 손해율 구간별 수수료율 매트릭스 |
| 이익수수료(Profit Commission) | 산식 (Profit × %), 관리비율(Management Expense %), 손실이월(Loss Carry Forward) 연수 |
| 중개수수료(Brokerage) | 중개사 개입 시 |

출처: [Munich Re Proportional Treaties](https://www.munichre.com/content/dam/munichre/contentlounge/website-pieces/documents/Proportional-Treaties-29-03-2023.pdf)

**(6) 적립금 및 이자 (Reserve Deposit)**
| 필드명 | 설명 |
|---|---|
| 보험료 적립금율(Premium Reserve %) | 통상 35~40% |
| 손해 적립금(Loss Reserve) | 미결손해 100% 등 |
| 적립금 이자율(Interest Rate %) | 연 환산 |
| 적립금 환급 시점 | 익기 / 익익기 |

**(7) 정산 조건 (Settlement Terms)**
| 필드명 | 설명 |
|---|---|
| 지급 기한 | 출재사 → 수재사: SOA 마감 후 N일 (통상 15일) |
| 확인 기한 | 수재사 → 출재사: SOA 수신 후 N일 (통상 14일) |
| 상계 허용(Offset) | 가능 / 불가 |
| Cash Loss 한도 | 즉시 청구 가능한 손해액 기준 |
| Portfolio Transfer | 보험료 % / 손해 % (Clean-Cut인 경우) |

출처: [Munich Re](https://www.munichre.com/content/dam/munichre/contentlounge/website-pieces/documents/Proportional-Treaties-29-03-2023.pdf), [LinkedIn Reinsurance Accounting](https://www.linkedin.com/pulse/reinsurance-accounting-proportional-treaties-basics-year-joseph)

#### 2.2.3 화면 액션
- `저장` / `임시저장`
- `복제`(전년도 특약 복사)
- `승인 요청` → 승인 후에만 거래입력 가능

### 2.3 [2단계] 출재/수재 명세 입력 화면

비례재보험은 정기적으로 **명세서(Bordereau)** 단위로 거래 데이터를 등록합니다.

#### 2.3.1 보험료 명세 입력 (Premium Bordereau)

**일괄 등록 / 개별 등록 모드 모두 지원** (엑셀 업로드 권장)

| 필드명 | 형식 | 필수 | 설명 |
|---|---|:---:|---|
| 특약 코드 | FK | ● | 등록된 특약에서 선택 |
| 회계 기간 | YYYYQN | ● | 예: 2026Q1 |
| 증권번호 | Text | ● | 원수계약 식별자 |
| 피보험자 | Text | ○ | |
| 위험 시작/종료일 | Date | ● | |
| 보험가입금액(Sum Insured) | Number | ● | 원화/외화 |
| 원보험료(Original Premium) | Number | ● | |
| 출재비율(%) | Number | ● | QS는 자동, Surplus는 위험별 산정 |
| 출재보험료(Ceded Premium) | Number | ● | 자동계산: 원보험료 × 출재비율 |
| 처리 구분 | Select | ● | 신규/취소/환급/조정 |
| 통화 | Select | ● | |

**자동 검증**:
- 특약 한도(Treaty Capacity) 초과 여부
- 보험기간이 특약기간 내 포함 여부
- 통화 일치 여부

#### 2.3.2 손해 명세 입력 (Loss Bordereau)

| 필드명 | 형식 | 필수 | 설명 |
|---|---|:---:|---|
| 특약 코드 | FK | ● | |
| 회계 기간 | YYYYQN | ● | |
| 사고번호(Claim No.) | Text | ● | |
| 연결 증권번호 | FK | ● | 보험료 명세에 등록된 증권 |
| 사고 발생일 | Date | ● | |
| 사고 보고일 | Date | ● | |
| 원수 지급보험금(Paid) | Number | ● | |
| 원수 미결손해(O/S Reserve) | Number | ● | |
| 출재비율(%) | Number | ● | 보험료 출재비율과 동일 |
| 재보험금 청구액(Recoverable) | Number | ● | 자동계산 |
| Cash Loss 여부 | Y/N | ● | 한도 초과 시 즉시 청구 |
| 손해 상태 | Select | ● | 진행중/지급완료/종결/거절 |

출처: [Munich Re Outstanding Loss Reserves](https://www.munichre.com/content/dam/munichre/contentlounge/website-pieces/documents/Proportional-Treaties-29-03-2023.pdf)

#### 2.3.3 입력 보조 기능
- 엑셀 템플릿 다운로드 / 업로드 (Bordereau 포맷)
- 행 단위 검증 결과 표시 (오류/경고)
- 임시저장 후 일괄 확정

### 2.4 [3단계] 기술계정 산출 화면

입력된 명세를 기준으로 **기술계정(Technical Account)**을 자동 산출합니다.

#### 2.4.1 산출 항목 (출재 기준)

```
[차변(Debit) - 수재사가 출재사에 줄 항목]
  + 출재수수료 (Ceding Commission)
  + 이익수수료 (Profit Commission)
  + 재보험금 (Claims Recoverable)
  + 적립금 환급 (Reserve Release)
  + 적립금 이자 (Interest on Deposit)

[대변(Credit) - 출재사가 수재사에 줄 항목]
  + 출재보험료 (Ceded Premium)
  + 적립금 신규 (Reserve Withheld)
  + 환급 보험료 (Return Premium)

[잔액 산출]
  Balance Due = Credit 합계 - Debit 합계
   - (+) 출재사 → 수재사 송금
   - (-) 수재사 → 출재사 송금
```

#### 2.4.2 화면 구성
- 기간 선택 (특약 + 회계기간)
- `기술계정 생성` 버튼 → 자동 계산
- 계산 결과 미리보기 (항목별 합계)
- 환율 적용 (다중통화일 경우 기준일 환율)
- 수동 조정 항목 추가 가능 (예: Portfolio Entry/Withdrawal)

출처: [LSI Basic Technical Reinsurance Accounting](https://elearning.londonschoolofinsurance.com/pdf/en/Basic_Technical_Reinsurance_Accounting.pdf)

---

### 2.5 [4단계] 계정명세서(SOA) 작성 화면

기술계정 결과를 표준 양식으로 출력하여 거래상대방에 송부할 SOA를 생성합니다.

#### 2.5.1 SOA 표준 구성
1. 표지: 특약명, 거래상대방, 회계기간, 통화
2. Premium Account: 보험료 명세 합계
3. Loss Account: 지급손해, 미결손해 변동
4. Commission Account: 출재/이익/중개수수료
5. Reserve Account: 적립금 신규/환급/이자
6. Cash Account: 잔액 및 정산 방향
7. 첨부: Premium / Loss Bordereau

#### 2.5.2 화면 기능
- PDF / Excel 출력
- 이메일 자동 발송 (거래상대방 담당자)
- 버전 관리 (Draft → Final → Confirmed)
- 첨부 자료 동시 송부

---

### 2.6 [5단계] 확인 및 승인 화면

#### 2.6.1 출재사(자사가 출재) 관점
- 송부한 SOA에 대한 수재사의 확인 응답 등록
- 차이 발생 시 조정 사유 입력
- 재발행(Re-issue) 또는 합의 처리

#### 2.6.2 수재사(자사가 수재) 관점
- 수신한 SOA 검증
- 자체 시스템 데이터와 대사(Reconciliation)
- 승인/이의제기/일부승인 처리

#### 2.6.3 상태 관리
```
Draft → Issued → Under Review → Confirmed → Settled
                      ↓
                  Disputed → Re-issued
```

### 2.7 [6단계] 송수신(자금 이체) 처리 화면

#### 2.7.1 송금 등록 (출재사 → 수재사)
| 필드 | 설명 |
|---|---|
| 정산 대상 SOA | 확정된 SOA 선택 |
| 송금 금액 | SOA 잔액 자동 |
| 송금 통화 | |
| 송금일 / 결제일 | |
| 송금 은행 / 계좌 | |
| 환율 (외화 시) | |
| 송금 증빙 첨부 | |

#### 2.7.2 수금 등록 (수재사 → 출재사)
- 입금 내역 매칭 (자동/수동)
- 미매칭 입금 보류 처리

#### 2.7.3 상계 처리 (Offset)
출재 SOA와 수재 SOA가 동일 상대방과 존재하는 경우, 특약 조건에 따라 상계 가능 ([Munich Re Offset Clause](https://www.munichre.com/content/dam/munichre/contentlounge/website-pieces/documents/Proportional-Treaties-29-03-2023.pdf)).

| 필드 | 설명 |
|---|---|
| 상계 대상 SOA 목록 | 다중 선택 |
| 상계 후 잔액 | 자동 계산 |
| 상계 통화 환산 | 기준 통화로 |
| 상계 승인자 | 결재 라인 |

---

### 2.8 [7단계] 미수금/미지급금 관리 화면

#### 2.8.1 채권채무 원장 (Subledger)
SOA가 확정되면 자동으로 채권/채무가 계상됩니다.

| 항목 | 발생 사유 |
|---|---|

## 3. 데이터 모델 (핵심 테이블)

```
[Treaty]              특약 마스터
  ├─ TreatyCondition  특약 조건 (출재율, 한도, 수수료 체계)
  └─ Commission       수수료 구조 (Sliding Scale 등)

[Counterparty]        거래상대방 마스터
[Currency / FXRate]   통화 / 환율

[Bordereau]
  ├─ PremiumBordereau 보험료 명세 (라인별)
  └─ LossBordereau    손해 명세 (라인별)

[TechnicalAccount]    기술계정 (산출 결과)
[StatementOfAccount]  SOA 헤더
  └─ SOA Line         SOA 라인 (항목별)

[Settlement]          송수금 거래
[Offset]              상계 처리

[ARLedger / APLedger] 미수/미지급 원장
[JournalEntry]        회계 분개

[ApprovalLog]         승인 이력
[AuditTrail]          변경 이력
```

## 4. 권한 및 결재 흐름

| 역할 | 권한 |
|---|---|

## 5. 개발 권장 순서 (Phased Delivery)

### Phase 1 — 기준정보 및 입력 (4–6주)
1. 거래상대방 / 통화 / 환율 마스터
2. 비례재보험 특약(Treaty) 등록 (Quota Share 우선)
3. 보험료 / 손해 Bordereau 입력 (엑셀 업로드 포함)

### Phase 2 — 정산 엔진 (4–6주)
4. 기술계정 자동 산출 (Quota Share)
5. SOA 작성 / 출력 / 송부
6. 확인·승인 워크플로

### Phase 3 — 자금 및 채권채무 (3–4주)
7. 송수금 등록 / 매칭
8. 미수금·미지급금 원장 및 Aging
9. 상계 처리

### Phase 4 — 확장 (4–6주)
10. Surplus 특약 지원 (위험별 출재율)
11. 이익수수료(Sliding Scale, Loss Carry Forward) 계산
12. Portfolio Transfer (Clean-Cut 방식)
13. 회계 자동분개 / IFRS 17 대응

### Phase 5 — 비비례재보험 (별도 트랙)
14. XOL / Stop Loss 특약 등록
15. Cash Call 청구 프로세스
16. Reinstatement Premium 처리

## 6. 핵심 비즈니스 규칙 체크리스트

- [ ] 특약 한도 초과 시 출재 불가 (Surplus의 경우)
- [ ] 보험료 출재비율과 보험금 출재비율 일치 검증
- [ ] 회계기간이 특약기간을 벗어나는 명세 차단
- [ ] 다중통화 거래 시 기준일 환율 일관 적용
- [ ] 적립금은 보험료의 일정율로 자동 계상, 익기 환급
- [ ] 이익수수료는 누적손익 기준, 손실이월 연수 반영
- [ ] Cash Loss 기준 초과 손해는 정기 SOA 외 별도 청구 가능
- [ ] 상계는 동일 통화/동일 거래상대방/특약 조항 허용 시에만
- [ ] SOA 확정 후 변경은 정정전표(Adjustment SOA) 발행
- [ ] 마감된 회계기간 변경은 차기로 조정 처리
