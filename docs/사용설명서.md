## 0. 시작하기 전에

### 0.1 이 앱은 누가 사용하나요?

| 역할                                    | 주요 작업                                                    |
| --------------------------------------- | ------------------------------------------------------------ |
| **브로커 실무자** (`broker_technician`) | 계약 등록, 명세 입력, 거래 확정, 정산서 초안 작성, 송금 처리 |
| **브로커 관리자** (`broker_manager`)    | 정산서 승인, 거래·명세 검수, 송금 검수                       |
| **검수자** (`reviewer`)                 | 거래·명세·정산서 더블체크 검수 (v1.5 신설)                   |
| **출재사 뷰어** (`cedant_viewer`)       | 발행된 정산서 열람 및 확인(Acknowledge)                      |
| **수재사 뷰어** (`reinsurer_viewer`)    | 발행된 정산서 열람 및 확인                                   |
| **시스템 관리자** (`admin`)             | 사용자 등록, 통화 마스터, 전체 권한                          |

### 0.2 8단계 프로세스 한눈에 보기

```
[1단계] 계약 등록 (Treaty Setup)
        ↓
[2단계] 명세 입력 (Bordereau — 보험료·손해 라인 명세)
        ↓
[3단계] 기술계정 산출 (Technical Account 자동 집계)
        ↓
[4단계] 정산서 작성 (Account Current — SOA Draft 생성)
        ↓
[5단계] 확인·승인 (Confirmation & Approval)
        ↓
[6단계] 정산 처리 (Settlement — 송수금 등록·매칭)
        ↓
[7단계] 미수·미지급금 관리 (AR/AP & Aging)
        ↓
[8단계] 회계 마감·보고 (Closing & Reporting)
```

> **앱 화면 왼쪽 사이드바**에 현재 작업 단계가 하이라이트되어 진행 상황을 확인할 수 있습니다.

## 1단계: 계약(특약) 등록

### 왜 이 단계가 필요한가?

계약(Treaty)은 이후 모든 정산 산출의 **기준 정보**입니다. 출재비율, 수수료율, 적립금율, 정산 주기가 모두 계약에서 자동으로 적용됩니다. 계약이 등록되어야 2단계 명세 입력이 가능합니다.

### 무엇을 입력하나?

| 섹션                | 주요 필드                                                             | 필수 |
| ------------------- | --------------------------------------------------------------------- | :--: |
| 기본 정보           | 계약번호, 계약유형(Treaty/Fac), Treaty 유형(비례/비비례), 보험종목    |  ●   |
| 거래상대방          | 출재사, 수재사 지분율(합계 100%)                                      |  ●   |
| 계약 기간·통화      | 개시일, 만기일, 정산통화, 정산주기                                    |  ●   |
| **항목별 정산주기** | 보험료 주기 / 보험금 주기 / 수수료 주기 (분기·반기·연간·수시 중 선택) |  ●   |
| 수수료 체계         | 출재수수료율(%), 이익수수료율(%), 중개수수료율(%)                     |  ○   |
| 적립금·이자         | 보험료 적립금율(%), 손해 적립금율(%), 이자율(%), 환급시점             |  ○   |
| 정산 조건           | 지급기한(일), 확인기한(일), Cash Loss 한도, 상계 허용, 인수 기준      |  ○   |

### 어떻게 입력하나? (단계별 클릭 경로)

1. 왼쪽 사이드바에서 **계약 관리 (1)** 클릭
2. 우측 상단 **+ 계약 등록** 버튼 클릭
3. **기본 정보** 섹션 입력 — 계약번호, 유형, 출재사, 개시/만기일, 통화, 정산주기
4. **지분율** 섹션에서 수재사별 지분율(%) 입력 — 합계 100%가 되어야 저장 가능
5. **(선택)** 아래쪽 접힌 섹션(**▶ 수수료 체계**, **▶ 적립금·이자**, **▶ 정산 조건**)을 클릭해 펼친 후 해당 조건 입력
   > 각 필드 오른쪽의 **?** 아이콘을 클릭하면 상세 설명 팝업이 표시됩니다
6. **계약 등록** 버튼 클릭

### 어느 화면에서?

**URL**: `/contracts/new`

![계약 등록 화면](screenshots/01-contracts-new.png)

_캡처 이미지는 실제 화면 촬영 후 이 경로에 저장_

### 다음 단계로 가는 조건

계약이 성공적으로 등록(status = `active`)되면 2단계 명세 입력에서 이 계약을 선택할 수 있습니다.

> **v1.5**: 계약 등록 후 상세 페이지의 **"정산일정"** 탭에서 항목별 정산주기 인스턴스(예: 2026Q1, 2026Q2 …)를 자동 생성할 수 있습니다. "인스턴스 자동 생성" 버튼을 클릭하면 계약기간 전체에 걸쳐 인스턴스가 자동으로 생성됩니다.

## 2단계: 명세 입력 (Bordereau)

### 왜 이 단계가 필요한가?

재보험 정산에서는 **원수계약 증권/사고 단위**로 출재 내역을 추적해야 합니다. 명세(Bordereau)를 입력하면 라인별 검증(계약기간 초과, 통화 불일치, 출재보험료 계산 오류 등)이 자동으로 이루어집니다.

> **거래 관리**에서는 집계 금액 단위로만 입력 가능합니다. **라인 단위 추적**이 필요한 경우 반드시 명세 입력을 사용하세요.

### 무엇을 입력하나?

#### 보험료 명세 (Premium Bordereau)

| 필드         | 설명                                              | 필수 |
| ------------ | ------------------------------------------------- | :--: |
| 계약         | 등록된 계약 선택                                  |  ●   |
| 회계기간     | 예: `2026Q1`(분기), `2026S1`(반기), `2026A`(연간) |  ●   |
| 증권번호     | 원수계약 식별자                                   |  ●   |
| 위험기간     | 시작일~종료일 (계약기간 내여야 함)                |  ●   |
| 보험가입금액 | Sum Insured                                       |  ●   |
| 원보험료     | 재보험 출재 전 원수보험료                         |  ●   |
| 출재비율(%)  | 30 또는 0.30 모두 입력 가능                       |  ●   |
| 출재보험료   | 원보험료 × 출재비율 (자동계산)                    |  ●   |
| 처리구분     | 신규 / 취소 / 환급 / 조정                         |  ●   |

#### 손해 명세 (Loss Bordereau)

| 필드                   | 설명                               | 필수 |
| ---------------------- | ---------------------------------- | :--: |
| 사고번호               | 클레임 식별자                      |  ●   |
| 사고 발생일 / 보고일   |                                    | ●/○  |
| 지급보험금 (Paid)      | 실지급액                           |  ○   |
| 미결손해 (O/S Reserve) | 예상 미지급액                      |  ○   |
| 재보험금 회수액        | (Paid + O/S) × 출재비율 (자동계산) |  ●   |
| Cash Loss 여부         | 계약의 Cash Loss 한도 초과 시 체크 |  ○   |
| 손해 상태              | 진행중 / 지급완료 / 종결 / 거절    |  ●   |

### 어떻게 입력하나?

#### 방법 1: 직접 입력

1. 사이드바 **명세 입력 (2)** 클릭
2. **+ 보험료 명세 추가** 또는 **+ 손해 명세 추가** 버튼 클릭
3. 각 필드 입력 — 출재보험료·재보험금 회수액은 자동계산됨
4. **저장** 클릭

#### 방법 2: CSV 일괄 업로드 (권장)

1. 사이드바 **명세 입력 (2)** 클릭
2. 우측 상단 **CSV 업로드** 버튼 클릭
3. 명세 유형(보험료/손해) 선택 → **샘플 CSV 다운로드** 클릭해 서식 확인
4. 데이터를 CSV에 입력하여 저장
5. 계약 선택 → CSV 파일 업로드 → **업로드** 버튼 클릭
6. 행별 검증 결과 확인 — 오류 행은 **오류 상태**로 저장되어 목록에서 확인 가능

### 다음 단계로 가는 조건

보험료 명세와 손해 명세의 검증 상태가 **정상(valid)** 또는 **경고(warning)**인 라인들이 입력되면 3단계로 진행합니다. **오류(error) 상태** 라인은 정산서 집계 전에 수정하세요.

> **v1.5 확정·검수**: 명세 상세 페이지에서 **[확정]** 버튼(브로커 실무자)과 **[검수 완료]** 버튼(검수자/관리자)을 순서대로 클릭하면 명세별 검토 이력(담당자명·이메일·일시)이 스냅샷으로 기록됩니다.
>
> **v1.5 정산주기 연결**: 명세 입력 시 계약 선택 후 정산주기 인스턴스(예: 2026Q1)를 선택하면 기간이 자동 채워지고 해당 인스턴스에 명세가 연결됩니다.

## 3~4단계: 기술계정 산출 & 정산서 작성 (SOA)

### 왜 이 단계가 필요한가?

2단계에서 입력된 보험료·손해 명세와 거래 내역을 바탕으로 **Account Current(정산서)**를 생성합니다. 이 정산서가 거래상대방(수재사)에게 발송되는 공식 청구서 역할을 합니다.

기술계정(Technical Account)은 정산서 생성 시 자동으로 집계됩니다:

```
[정산서 집계 항목]
  출재보험료 (Credit)          ↑ 수재사가 받을 항목
  출재수수료 (Debit)           ↓ 출재사가 받을 항목
  재보험금 회수 (Debit)        ↓ 출재사가 받을 항목
  보험료 적립금 신규 (Credit)  ↑ 수재사 유보
  보험료 적립금 환급 (Debit)   ↓ 이전 기간 적립금 반환
  이자 (Debit)                 ↓ 출재사가 받을 항목
  이월잔액 (Balance B/F)       자동 계산

  ──────────────────────────────
  순 정산금액 (Net Balance)
  (+) 출재사 → 수재사 지급
  (-) 수재사 → 출재사 지급
```

### 무엇을 입력하나?

| 필드      | 설명                           |
| --------- | ------------------------------ |
| 계약      | 정산 대상 계약 선택            |
| 수재사    | 이 정산서를 받을 수재사 선택   |
| 정산 기간 | 정산 기간 유형 + 시작일/종료일 |
| 정산 통화 | 기본값은 계약의 정산통화       |
| 비고      | 특이사항                       |

### 어떻게 하나?

1. 사이드바 **정산서 관리 (4)** 클릭
2. **+ 정산서 생성** 버튼 클릭
3. 계약과 수재사, 정산 기간 입력
4. **중복 기간 경고**가 표시되면 기존 정산서를 확인하고 진행
5. **생성** 버튼 클릭 → 시스템이 해당 기간의 거래를 자동 집계
6. 생성된 정산서 상세 페이지에서 집계 금액 및 항목별 내역 확인

| 정산서 생성 | `/account-currents/new`  |
| 정산서 상세 | `/account-currents/{id}` |

정산서 status가 **초안(draft)**이면 내용 수정이 가능합니다. 내용이 확인되면 **승인 요청** 버튼으로 5단계로 진행합니다.

## 5단계: 확인 및 승인

### 왜 이 단계가 필요한가?

정산서는 **내부 승인 → 발행 → 외부 확인** 세 단계를 거쳐야 정식으로 효력이 발생합니다. 이를 통해 금액 오류 발생 시 사전에 수정할 수 있습니다.

### 정산서 상태 흐름 (v1.5 — 6단계)

```
초안(Draft)
  → 승인대기(Pending Approval)   [승인 요청 버튼 — 브로커 실무자]
  → 승인됨(Approved)             [승인 버튼 — 브로커 관리자]
  → 검수됨(Reviewed)             [검수 버튼 — reviewer / 관리자]  ← v1.5 신설
  → 발행됨(Issued)               [발행 버튼 — 브로커 관리자]
  → 확인됨(Acknowledged)         [수재사 뷰어가 확인]
       ↓ (이의 있는 경우)
  → 이의신청(Disputed)
       ↓
  → 취소(Cancelled) + 새 정산서 재발행
```

> 각 단계는 정산서 상세 페이지 상단의 **승인 스텝퍼**에서 현재 진행 상황을 한눈에 확인할 수 있습니다.

### 거래·명세 확정·검수 흐름 (v1.5 신설)

거래(Transaction)와 명세(Bordereau)도 별도의 확정·검수 단계를 거칩니다:

```
미확정(Unconfirmed)
  → 담당자 확정(Confirmed)   [확정 버튼 — 브로커 실무자 이상]
  → 검수 완료(Verified)      [검수 완료 버튼 — reviewer / 관리자]
       ↓ (반려 시)
  → 반려(Rejected)           [반려 + 반려 사유 입력]
```

> 확정·검수 담당자명, 이메일, 처리 일시가 스냅샷으로 기록되어 추적 가능합니다.

### 브로커(내부) 담당자가 할 일

1. 정산서 상세 페이지 우측 상단 **승인 요청** 버튼 클릭
2. 관리자가 내용 검토 후 **승인** 버튼 클릭
3. 검수자 또는 관리자가 **검수** 버튼 클릭 (v1.5)
4. 검수 후 **발행** 버튼 클릭 → 수재사/출재사에게 공유 링크(토큰 URL) 발송

### 외부 뷰어(수재사)가 할 일

1. 이메일로 받은 공유 링크 접속 (별도 로그인 불필요)
2. 정산서 내용 검토
3. **확인(Acknowledge)** 버튼 클릭 → status가 `acknowledged`로 변경
4. 이의가 있는 경우 **이의신청(Dispute)** 버튼 클릭

| ----------------------- | --------------------------------- |

| 외부 뷰어 화면          | `/external/account-currents/{id}` |
| 공유 링크               | `/share/{token}`                  |

## 6단계: 정산 처리 (송수금)

### 왜 이 단계가 필요한가?

정산서가 확인(Acknowledged)되면 실제 자금 이동이 일어납니다. **송금** 또는 **수금** 내역을 등록하고 정산서와 **매칭**해야 미수·미지급금이 청산됩니다.

### 무엇을 입력하나?

| 필드       | 설명                          |
| ---------- | ----------------------------- |
| 정산 유형  | 수금(Receipt) / 송금(Payment) |
| 거래상대방 | 송금 또는 수금 상대방         |
| 금액·통화  |                               |
| 정산일     | 실제 입금/출금일              |
| 메모       | 은행 참조번호 등              |

### 어떻게 하나?

1. 사이드바 **결제 관리 (6)** 클릭
2. **+ 결제 등록** 버튼 클릭 → 수금/송금 등록
3. 결제번호 링크 클릭 → **결제 상세** 페이지에서 매칭된 정산서 및 송금 상태 확인
4. 목록의 **매칭 패널** 버튼 → 드래그앤드롭으로 정산서와 매칭
5. 매칭 후 결제 상세에서 **[송금 완료]** 버튼 클릭 (브로커 실무자)
6. **[송금 검수]** 버튼 클릭 (검수자/관리자) — v1.5 신설

| 화면                  | URL                 |
| --------------------- | ------------------- |
| 결제 목록             | `/settlements`      |
| 결제 상세 + 송금 처리 | `/settlements/{id}` |

## 7단계: 미수·미지급금 관리

### 왜 이 단계가 필요한가?

정산서가 발행되었지만 자금이 아직 이동되지 않은 금액, 즉 **미청산 잔액(Outstanding)**을 추적합니다. 경과일수(Aging)별로 분류하여 연체 현황을 파악합니다.

| 컬럼 | 설명 |
|---|---|
| 거래상대방 | |
| 계약 / SOA 번호 | |
| 발생일 / 만기일 | |
| 금액 | |
| Aging 구간 | 0~30일 / 31~60일 / 61~90일 / 91~180일 / 180일+ |
| 상태 | 정상 / 연체 / 분쟁 |

### 어떻게 보나?

3. **Aging 구간 필터** 선택 (Current / 1-30일 / 31-60일 / 61-90일 / 90일 초과) — v1.5 신설

## 부록 C: 자주 묻는 질문 (FAQ)

**Q. 계약 저장 시 "지분율 합계가 100%이어야 합니다" 오류가 납니다.**

> 지분율 섹션에서 모든 수재사의 지분율(%)을 더하면 정확히 100.00이어야 합니다. 소수점 오차가 있을 경우 소수점 4자리까지 조정하세요.

**Q. 명세 입력 시 "통화 불일치" 오류가 납니다.**

> 명세에 입력한 통화가 해당 계약의 정산통화와 다릅니다. 계약의 **계약 관리** 페이지에서 정산통화를 확인한 후 명세의 통화를 동일하게 수정하세요.

**Q. 정산서가 "이의신청(Disputed)" 상태가 되었습니다. 어떻게 처리하나요?**

> 정산서 상세 페이지에서 **취소** 버튼으로 원본 정산서를 취소하면 연결 거래의 잠금이 해제됩니다. 이후 **+ 정산서 생성**에서 수정된 금액으로 새 정산서를 발행하세요.

**Q. 외부 뷰어(수재사)가 링크에 접속했는데 화면이 안 보입니다.**

> 공유 링크(토큰 URL)는 **발행일로부터 30일간 유효**합니다. 만료된 경우 정산서 상세 페이지의 **공유 링크 관리** 패널에서 새 링크를 발행하세요.

**Q. CSV 업로드 시 "파싱 오류" 메시지가 납니다.**

> 샘플 CSV를 다운로드하여 **컬럼 이름과 순서**가 일치하는지 확인하세요. 숫자 필드에 쉼표(,)가 포함되면 오류가 발생합니다. CSV를 UTF-8 형식으로 저장하세요.

**Q. 같은 기간에 정산서를 두 번 만들 수 있나요?**

> 동일 계약·수재사·기간에 대해 중복 정산서를 생성하면 **중복 경고**가 표시됩니다. 기존 정산서가 취소(Cancelled) 상태가 아니라면 새 정산서 생성이 차단됩니다.

**Q. 이익수수료(Profit Commission)는 어떻게 계산되나요?**

> 계약에 이익수수료율(%)만 설정 가능합니다. 이익수수료는 **거래 관리** > **거래 등록**에서 `commission` 유형으로 수동 입력하세요.

**Q. 검수자(reviewer)가 확정/검수 버튼을 볼 수 없습니다.**

> 관리자 페이지(`/admin`)에서 해당 사용자의 역할이 `reviewer`로 설정되어 있는지 확인하세요. 역할 변경 후 재로그인이 필요할 수 있습니다.

**Q. 정산주기 인스턴스를 잘못 생성했습니다. 어떻게 삭제하나요?**

> 계약 상세 페이지의 **"정산일정"** 탭에서 각 인스턴스 행을 선택해 개별 삭제가 가능합니다. 이미 명세가 연결된 인스턴스는 삭제 전 명세 연결을 해제해야 합니다.

**Q. 보험금 청구(Claims) 기능은 어떻게 사용하나요?**

> 사이드바 **청구 관리** 메뉴에서 사고 건별 청구 헤더를 등록하고, 상세 페이지에서 수재사 수금 거래와 출재사 지급 거래를 연결합니다. 수금/지급 합계가 자동으로 집계되어 사고 건별 잔액을 추적할 수 있습니다.
